W3C Digital Credentials API + ISO mdoc

SMART Health Check-in over direct mdoc

This prototype shows a web verifier asking an Android wallet for patient-directed clinical data using a SMART Health Check-in request embedded in an org-iso-mdoc Digital Credentials API flow.

Open verifier demo Open protocol explainer

What this demonstrates

Verifier request

The browser creates a direct mdoc request with docType = org.smarthealthit.checkin.1. The SMART request JSON lives in ItemsRequest.requestInfo["org.smarthealthit.checkin.request"].

Wallet consent

The Android wallet parses the request, matches supported items, renders a consent UI, and returns artifacts such as FHIR JSON, QuestionnaireResponse resources, or SMART Health Cards.

Encrypted response

The wallet places the SMART response JSON in the stable mdoc element smart_health_checkin_response, signs the mdoc response, and encrypts it back to the verifier with HPKE.

Use the two hosted entry points

Verifier demo

A static relying-party web app that builds the request, invokes navigator.credentials.get(), opens the wallet response, and validates artifacts against the original request.

It needs an HTTPS context and a browser/device stack that supports the direct org-iso-mdoc Digital Credentials API path.

Protocol explainer

A byte-level walkthrough using checked-in Chrome/Android capture fixtures. It annotates the request, response, SessionTranscript, HPKE envelope, mdoc tags, and decoded SMART payloads.

The fixture tree is included in the Pages artifact at ./fixtures/ so the explainer can load the same files used by the test suites.
