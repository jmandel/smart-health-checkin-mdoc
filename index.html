What this is

SMART Health Check-in defines an application-level request and response shape for patient-directed clinical sharing. The request is a list of consentable items such as demographics, insurance, clinical summaries, or intake questionnaires. The response returns artifacts that fulfill those items.

The demo carries that request/response model through the browser's W3C Digital Credentials API, so the verifier can ask a wallet and receive an encrypted response without a server round trip.
