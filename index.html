Patient-directed clinical data exchange

SMART Health Check-in over W3C Digital Credentials API

This prototype shows a web verifier asking an Android wallet for patient-directed clinical data using a SMART Health Check-in request carried through the browser's Digital Credentials API.

Open verifier demo Open SMART model explainer

What this demonstrates

Verifier request

A clinic-style web page builds a SMART Health Check-in request: a purpose, a list of consentable items, and the artifact formats it can accept.

Wallet consent

The Android wallet parses the request, matches supported items, renders a consent UI, and returns artifacts such as FHIR JSON, QuestionnaireResponse resources, or SMART Health Cards.

Encrypted response

The wallet returns a SMART response with shared artifacts and a per-item status list, encrypted back to the verifier through the Digital Credentials API flow.

Use the hosted entry points

Verifier demo

A static relying-party web app that builds the request, invokes navigator.credentials.get(), opens the wallet response, and validates artifacts against the original request.

It needs an HTTPS context and a browser/device stack that supports the Digital Credentials API profile used by this prototype.

SMART model explainer

A readable walkthrough of the requester role, responder role, request item language, response artifacts, and per-item status model.

Wire protocol details

A byte-level companion for implementers. It uses the checked-in Chrome/Android fixtures to annotate the request, response, SessionTranscript, HPKE envelope, and technical carrying layer.
