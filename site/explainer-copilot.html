It decodes the CBOR, shows the bytes, and names the calculations that make the response verifier-bound and tamper-evident.

Protocol map

Verifier builds a request The web app creates a SMART request JSON object, embeds it inside an mdoc ItemsRequest, and wraps that in a direct org-iso-mdoc Digital Credentials API request.
Browser supplies the origin Credential Manager hands the wallet both the request JSON and CallingAppInfo.origin. The wallet must use this exact origin in the SessionTranscript.
Wallet parses and renders Android decodes deviceRequest, pulls requestInfo.smart_health_checkin, fetches any referenced questionnaires, and renders consent.
Wallet creates an mdoc response The selected SMART/FHIR payload is placed in org.smarthealthit.checkin.smart_health_checkin_response, digested in the MSO, signed by issuer and device keys, and HPKE-encrypted to the verifier key.
Verifier opens and checks The verifier uses the request's recipient private key and the exact SessionTranscript to open the HPKE ciphertext and inspect the mdoc DeviceResponse.

Request bytes: browser JSON -> mdoc DeviceRequest

The request contains two base64url strings. deviceRequest is the mdoc request CBOR. encryptionInfo is the verifier's HPKE recipient key and a nonce. Both strings are visible to the matcher and wallet; the verifier origin is not in the request payload and must come from Credential Manager.
DeviceRequest = CBOR({ "version": "1.0", "docRequests": [{ "itemsRequest": tag24(CBOR({ "docType": "org.smarthealthit.checkin.1", "nameSpaces": {"org.smarthealthit.checkin": {"smart_health_checkin_response": false}}, "requestInfo": {"smart_health_checkin": JSON.stringify(SMART request)} })) }] }) EncryptionInfo = CBOR(["dcapi", { "nonce": 32 random bytes, "recipientPublicKey": COSE_Key(P-256 verifier public key) }])

Response bytes: Android wallet -> encrypted DeviceResponse

The wallet response is JSON at the Credential Manager boundary, but the meaningful payload is again base64url CBOR: ["dcapi", {"enc": ..., "cipherText": ...}]. The ciphertext decrypts to an ISO mdoc DeviceResponse.
IssuerSignedItem = { "digestID": 0, "random": 16 bytes, "elementIdentifier": "smart_health_checkin_response", "elementValue": JSON.stringify(SMART response) } issuerSignedItemTag24 = tag24(CBOR(IssuerSignedItem)) valueDigest = SHA-256(issuerSignedItemTag24) MSO.valueDigests["org.smarthealthit.checkin"][0] = valueDigest issuerAuth = COSE_Sign1(payload = tag24(CBOR(MSO)), signer = issuer key) DeviceAuthentication = tag24(CBOR([ "DeviceAuthentication", SessionTranscript, "org.smarthealthit.checkin.1", tag24(CBOR(DeviceNameSpaces)) ])) deviceSignature = COSE_Sign1(payload = DeviceAuthentication, signer = MSO.deviceKeyInfo.deviceKey) dcapi response = CBOR(["dcapi", { "enc": ephemeral P-256 public key, "cipherText": HPKE-Seal(DeviceResponse, info = SessionTranscript) }])
