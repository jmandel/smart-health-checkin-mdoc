Patient-directed clinical data exchange

SMART Health Check-in over W3C Digital Credentials API

A web verifier asks a patient wallet for check-in information: FHIR resources, questionnaires, SMART Health Cards, and per-item status in a user-approved response.

One presentation model

SMART Health Check-in defines an application-level request and response shape for patient-directed clinical sharing. The request is a list of consentable items such as demographics, insurance, clinical summaries, intake questionnaires, SMART Health Cards, or other FHIR resources. The response returns artifacts that fulfill those items.

The demo carries that request/response model through the browser's W3C Digital Credentials API, so the verifier can ask a wallet and receive an encrypted response without a server round trip.

Flows and layers

Both entry points start from the same SMART request/response model, then carry it through the browser Digital Credentials API. The kiosk path adds one handoff layer above the shared stack.
